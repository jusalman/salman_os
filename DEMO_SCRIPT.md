# SALMAN OS Demo Script

## Demo Goal

SALMAN OS v1 데모의 목표는 내부 직원이 고객사 운영 정보를 한 화면에서 빠르게 파악하고 다음 행동을 결정할 수 있음을 보여주는 것이다.

## Opening

SALMAN OS는 Google Sheets, Google Drive, Google Calendar를 대체하는 제품이 아니다. 고객사별 자료, 업무, 일정, 비즈머니, 링크, 로그를 한 화면에서 연결해 보여주는 내부 고객사 운영센터 MVP다.

v1은 내부 직원용이며, 외부 고객 포털은 제공하지 않는다.

## Login Flow

1. SALMAN OS 접속 화면을 연다.
2. 공통 비밀번호 `salman1!`을 입력한다.
3. 본인 이름을 입력한다.
4. 내부 운영 화면으로 진입한다.

데모 중 직원별 계정, 이메일 인증, 고객 초대 기능은 v1 범위가 아니라고 설명한다.

## Main Scenario

### 1. 고객사 선택

고객사 목록에서 데모 고객사를 선택한다. 화면에는 해당 고객사의 기본 상태, 담당자, 최근 운영 로그가 보인다.

### 2. 자료 확인

자료 영역에서 Google Drive 원본 파일 링크를 확인한다. SALMAN OS는 파일 저장소가 아니라 Drive 원본 링크와 운영 맥락을 연결하는 화면임을 설명한다.

### 3. 업무 확인

업무 영역에서 진행 중 업무, 지연 업무, 완료 업무를 확인한다. 업무는 고객사 기준으로 연결되어 있고 담당자와 마감일을 가진다.

### 4. 일정 확인

일정 영역에서 고객사별 내부 일정을 확인한다. 이 일정은 Google Calendar 연동이 아니라 Supabase 기반 내부 일정 데이터다.

고객사별 실제 Google Calendar는 생성하지 않는다고 명확히 말한다.

### 5. 비즈머니 확인

비즈머니 영역에서 확인 필요 상태와 관련 링크를 본다. v1에서는 자동 수집하지 않고, 직원이 확인한 상태와 로그를 남긴다.

### 6. Smart Operation Views

Smart Operation Views에서 마감 임박, 확인 필요, 최근 변경 항목을 확인한다. 이 기능은 AI가 아니라 템플릿 기반 필터와 정렬이다.

### 7. 파일 보관 처리

더 이상 쓰지 않는 파일은 삭제하지 않고 Google Drive의 `99_Archive`로 보관 처리한다. SALMAN OS에는 보관 상태와 로그가 남는다.

## Do Not Demo

v1 데모에서 다음 기능은 보여주지 않는다.

- OpenAI API 응답
- AI 직원 실행
- 비즈머니 자동 수집
- Playwright 자동화
- Google Calendar 연동
- 외부 고객 포털

## Closing

SALMAN OS v1은 자동화 완성 제품이 아니라 내부 직원이 고객사 운영 정보를 잃지 않고 한 화면에서 연결해 보기 위한 MVP다. AI 직원 기능은 v1.5 이후 운영 패턴이 검증된 뒤 활성화한다.
