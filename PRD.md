# SALMAN OS PRD

## Summary

SALMAN OS는 내부 직원용 고객사 운영센터 MVP다. 고객사별 자료, 업무, 일정, 비즈머니, 링크, 로그를 한 화면에서 연결해 보여주는 것이 핵심이다.

SALMAN OS는 Google Sheets, Google Drive, Google Calendar를 대체하는 시스템이 아니다. 이미 존재하는 원본 자료와 운영 정보를 고객사 기준으로 묶어 내부 직원이 빠르게 확인하도록 돕는다.

## Problem

고객사 운영 정보가 여러 곳에 흩어져 있으면 내부 직원은 다음 문제를 겪는다.

- 고객사별 최신 자료를 찾기 어렵다.
- 업무와 일정이 분리되어 맥락을 놓친다.
- 비즈머니 확인 상태가 사람마다 다르게 관리된다.
- 중요한 링크와 운영 로그가 대화방이나 개인 메모에 흩어진다.
- 고객사 상태를 한 화면에서 설명하기 어렵다.

## Goal

v1의 목표는 내부 직원이 SALMAN OS에서 고객사를 선택하면 해당 고객사의 자료, 업무, 일정, 비즈머니, 링크, 로그를 한 번에 확인할 수 있게 하는 것이다.

성공 기준:

- 고객사별 운영 상태를 한 화면에서 파악할 수 있다.
- Google Drive 원본 자료로 빠르게 이동할 수 있다.
- Supabase 기반 내부 일정과 업무를 함께 확인할 수 있다.
- 비즈머니 확인 필요 상태를 놓치지 않는다.
- 운영 변경 사항이 로그로 남는다.

## Users

v1 사용자는 내부 직원이다.

포함:

- 고객사 운영 담당자
- 내부 관리자
- 고객사 자료와 업무를 확인해야 하는 팀원

제외:

- 외부 고객
- 고객사 담당자
- 파트너 계정

## Access

v1 접속 방식은 단순 MVP 기준으로 운영한다.

- 공통 비밀번호: `salman1!`
- 본인 이름 입력

직원별 계정, 이메일 로그인, 권한 그룹, 외부 고객 초대는 v1 범위가 아니다.

## Core Features

### Client Operation Center

고객사를 선택하면 해당 고객사의 운영 정보를 한 화면에서 보여준다.

- 기본 정보
- 담당자
- Google Drive 루트 링크
- 최근 업무
- 내부 일정
- 비즈머니 상태
- 주요 링크
- 운영 로그

### Files

Google Drive는 원본 파일 기준이다. SALMAN OS는 파일을 저장소처럼 대체하지 않고, 고객사별 원본 파일 링크와 상태를 관리한다.

파일 삭제는 영구 삭제가 아니라 `99_Archive` 보관 처리로 정의한다.

### Tasks

고객사별 업무를 상태, 담당자, 마감일 기준으로 관리한다.

### Calendar

v1 캘린더는 Google Calendar 연동이 아니다. Supabase 기반 내부 고객사별 일정 관리다.

고객사별 실제 Google Calendar는 생성하지 않는다.

### Business Money

비즈머니는 자동 수집하지 않는다. v1에서는 확인 링크, 확인 상태, 마지막 확인일, 메모를 관리한다.

### Smart Operation Views

Smart Operation Views는 AI 없이 템플릿 기반으로 제공한다.

예시:

- 오늘 확인할 고객사
- 마감 임박 업무
- 확인 필요 비즈머니
- 최근 보관 처리된 파일
- 최근 운영 로그

## Out of Scope

v1에서는 다음을 제외한다.

- OpenAI API
- AI 직원 실행
- 비즈머니 자동 수집
- Playwright 자동화
- Google Calendar 연동
- 고객사별 실제 Google Calendar 생성
- 외부 고객 포털

AI 직원 기능은 v1.5 이후 활성화 예정이다.
