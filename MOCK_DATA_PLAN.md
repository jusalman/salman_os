# SALMAN OS Mock Data Plan

## Purpose

이 문서는 SALMAN OS v1 MVP 화면과 데모를 검증하기 위한 목업 데이터 기준을 정의한다. 목업 데이터는 내부 직원용 운영센터 경험을 보여주는 데 집중한다.

## Mock Data Goals

- 고객사별 자료, 업무, 일정, 비즈머니, 링크, 로그가 한 화면에서 연결되는지 확인한다.
- Supabase 운영 데이터 구조가 화면 요구사항을 감당하는지 확인한다.
- Google Drive 원본 파일 링크가 운영 맥락과 자연스럽게 연결되는지 확인한다.
- Smart Operation Views가 AI 없이 템플릿 기반으로 유용한지 확인한다.

## Clients

최소 3개 고객사를 준비한다.

- 활성 고객사: 업무와 일정이 모두 있는 상태
- 확인 필요 고객사: 비즈머니 또는 마감 업무가 있는 상태
- 보관 고객사: 일부 자료가 `99_Archive` 처리된 상태

## Staff Members

내부 직원 이름 3명을 준비한다. v1 로그인은 공통 비밀번호 `salman1!`과 본인 이름 입력 기준이다.

목업 데이터에는 이메일, 고객 계정, 외부 권한 정보를 넣지 않는다.

## Tasks

고객사별로 다음 업무 상태를 포함한다.

- 진행 중
- 지연
- 확인 필요
- 완료
- 보관됨

업무는 담당자, 마감일, 관련 자료 링크를 가질 수 있다.

## Events

일정 데이터는 Supabase 기반 내부 일정으로 만든다.

- 오늘 일정
- 이번 주 일정
- 완료된 일정
- 취소된 일정

Google Calendar 이벤트 ID, Google Calendar 토큰, 고객사별 실제 Google Calendar 정보는 만들지 않는다.

## Files

파일 데이터는 Google Drive 원본 링크를 가진 메타데이터로 만든다.

- 계약서 PDF
- 광고 소재 이미지
- 월간 보고서
- 업무 요청 문서
- `99_Archive`에 보관된 이전 버전

실제 파일 업로드 자동화나 Drive 자동 정리는 v1 범위가 아니다.

## Business Money

비즈머니 데이터는 자동 수집 결과가 아니라 직원 확인 상태로 만든다.

- 확인 필요
- 정상 확인
- 이슈 있음
- 마지막 확인일 없음

각 항목은 확인 링크와 메모를 가진다.

## Operation Logs

고객사별 최근 로그를 준비한다.

- 업무 생성
- 일정 수정
- 파일 보관 처리
- 비즈머니 확인
- 내부 메모 추가

## Excluded Mock Data

다음 데이터는 만들지 않는다.

- OpenAI API 응답
- AI 직원 실행 결과
- Playwright 자동화 로그
- Google Calendar 연동 토큰
- 외부 고객 포털 계정
